# Secure Web Chat Ecosystem

学習用の「攻防一体型」チャットアプリケーションです。
単なるチャット機能だけでなく、セキュリティの脆弱性と、それに対する堅牢な実装を体系的に学習することを目的としています。

## 🛠 現在の進捗
- [x] **フェーズ1**: FastAPI + WebSocket によるリアルタイム通信の基盤実装
- [x] **フェーズ2**: SQLite + プリペアドステートメントによる認証の基礎
- [x] **フェーズ3**: パスワードのハッシュ化（bcryptによる保存の安全性向上）
- [x] **フェーズ4**: セッション管理（Cookieによるログイン状態の維持とログアウト）
- [x] **フェーズ5**: セキュリティ・ラボ（SQLインジェクションの攻防体験モード）
- [x] **フェーズ6**: AI監視エンジンによるリアルタイム攻撃検知（正規表現・正規化対応）
- [x] **フェーズ7**: 管理者用マルチモニタリング・ダッシュボード（基盤実装） ✅ NEW
- [ ] **フェーズ8**: スクリーンキャプチャ通信による実時間監視（実装中）

## 🚀 アップデート内容 (ver 0.7)
- **管理者用マルチモニタリングパネル**:
    - **独立型ダッシュボード**: 一般UIの制約を受けない独立した管理者専用画面（`/admin`）を構築。
    - **インフラステータス監視**: 全体の接続人数と、AIが阻止した攻撃の累計数をリアルタイムに同期表示。
    - **4分割スロット・レイアウト**: CSS Gridを用いた2x2の監視窓を実装。特定のユーザーを選択して監視対象に割り当てるスロット管理機能を搭載。
- **フロントエンドのリファクタリング**:
    - **ESモジュール化**: `type="module"` を導入し、機能ごとにJSファイルを分離（`network.js`, `admin.js`, `admin_app.js`）。
    - **疎結合な設計**: 通信基盤（`network.js`）とUI操作を分離し、メンテナンス性を向上。
- **WebSocketハンドリングの強化**:
    - Python側での `user_map` 管理により、接続中のユーザー名を正確にブロードキャストする仕組みを実装。

## 📂 ディレクトリ構成
- `app/ai/`: `detector.py`（AI監視エンジン）
- `app/core/`: 
    - `websocket_handler.py`: 通信制御。インフラ状態のブロードキャスト。
    - `auth.py`, `database.py`: 認証とDB管理
- `app/static/`:
    - `css/`: `admin.css`（2x2グリッドレイアウト）
    - `js/`: 
        - `network.js`: 通信基盤。複数コールバック（Chat/Alert/Reset/Infra）に対応。
        - `admin.js`: 管理画面のUI更新、スロット割り当てロジック。
        - `admin_app.js`: 管理画面のメインエントリーポイント。
- `app/templates/`: `admin.html`（独立した管理者UI）
