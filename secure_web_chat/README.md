# Secure Web Chat Ecosystem (ver 0.8)

学習用の「攻防一体型」チャットアプリケーションです。
単なるチャット機能だけでなく、セキュリティの脆弱性と、それに対する堅牢な実装を体系的に学習することを目的としています。

## 🛠 現在の進捗
- [x] **フェーズ1**: FastAPI + WebSocket によるリアルタイム通信の基盤実装
- [x] **フェーズ2**: SQLite + プリペアドステートメントによる認証の基礎
- [x] **フェーズ3**: パスワードのハッシュ化（bcryptによる保存の安全性向上）
- [x] **フェーズ4**: セッション管理（Cookieによるログイン状態の維持とログアウト）
- [x] **フェーズ5**: セキュリティ・ラボ（SQLインジェクションの攻防体験モード）
- [x] **フェーズ6**: AI監視エンジンによるリアルタイム攻撃検知（正規表現・正規化対応）
- [x] **フェーズ7**: 管理者用マルチモニタリング・ダッシュボード（基盤実装）
- [x] **フェーズ8**: スクリーンキャプチャ通信による実時間監視（実装済み） ✅ **UPDATE**
- [ ] **フェーズ9**: 管理者からユーザーへの動的命令（キック・警告表示）の実装

## 🚀 アップデート内容 (ver 0.8)
- **実時間スクリーンモニタリング**:
    - **WebRTC/Canvas統合**: `getDisplayMedia` を利用し、ユーザーのブラウザ画面を動的にキャプチャ。
    - **インテリジェント・スロット管理**: `admin.js` による動的な `<img>` 生成と、CSS `object-fit: contain` によるアスペクト比を維持した4分割表示。
    - **適応型サンプリング**: 通信負荷と判読性のバランスを考慮した「1024px / 8秒間隔 / 50% JPEG圧縮」の設定を導入。
- **UI/UXの改善**:
    - **LIVEラベル表示**: 監視スロットに「LIVE: {user}」を絶対位置（`z-index`）で表示し、多人数監視時の視認性を向上。
    - **NO SIGNAL状態の管理**: 未接続スロットの視覚的フィードバックの実装。

## 🔬 技術的ハイライト (Lessons Learned)
- **セルフキャプチャ回避**: Chromeの仕様による自タブ非表示を、別タブ（Target/Sender）構成によるテスト手法で解決。
- **再帰的無限ループ対策**: 管理画面を共有した際の「合わせ鏡現象」の理解と、タブ単位の共有による回避策の確立。
- **負荷分散**: 高解像度（1024px）を維持しつつ、送信頻度を落とすことで、低速なアップロード環境でも安定した監視を可能にした。

        - `monitor.js`: 画面キャプチャ・Base64変換・タイマー送信。
        - `admin.js`: モニターUIの動的更新、ラベル制御。
- `app/templates/`: `admin.html`
