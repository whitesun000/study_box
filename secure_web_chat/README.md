# Secure Web Chat Ecosystem

学習用の「攻防一体型」チャットアプリケーションです。
単なるチャット機能だけでなく、セキュリティの脆弱性と、それに対する堅牢な実装を体系的に学習することを目的としています。

## 🛠 現在の進捗
- [x] **フェーズ1**: FastAPI + WebSocket によるリアルタイム通信の基盤実装
- [x] **フェーズ2**: SQLite + プリペアドステートメントによる認証の基礎
- [x] **フェーズ3**: パスワードのハッシュ化（bcryptによる保存の安全性向上）
- [x] **フェーズ4**: セッション管理（Cookieによるログイン状態の維持とログアウト）
- [x] **フェーズ5**: セキュリティ・ラボ（SQLインジェクションの攻防体験モード）
- [x] **フェーズ6**: AI監視エンジンによるリアルタイム攻撃検知（正規表現・正規化対応） ✅ DONE
- [ ] **フェーズ7**: 攻撃検知時の自動防御（ペイロードの無効化）

## 🚀 アップデート内容 (ver 0.6)
- **AI監視エンジン (AIDetector)**:
    - 正規表現によるシグネチャベースの攻撃検知を実装。
    - **難読化対策**: メッセージから記号を除去して解析する「正規化プロセス」を導入。`S/E/L/E/C/T` のような回避手法も検知可能。
    - `CRITICAL`（攻撃構文）、`WARNING`（不審なワード）、`MONITORING`（特定ユーザー）の3段階アラート。
- **WebSocket通信の構造化 (JSON Protocol)**:
    - 通信を単純な文字列からJSON形式にリファクタリング。
    - チャット送信、AIアラート受信、システム操作を同一ルート内で識別可能に。
- **管理者用チャットリセット**:
    - 全接続ユーザーの画面を同期してクリアする一斉リセット機能を実装。

## 📂 ディレクトリ構成
- `app/ai/`:
    - `detector.py`: AI監視エンジン。正規表現と正規化による攻撃解析。
- `app/core/`: 
    - `auth.py`: 認証ロジック
    - `websocket_handler.py`: 通信制御。JSONデコードによる命令分岐（New!）
    - `database.py`: DB接続管理
- `app/templates/`: UI（HTML）
- `static/js/`: 
    - `main.js`: チャットUI制御
    - `network.js`: WebSocket基盤（JSON stringify/parse対応）
    - `lab.js`: セキュリティ・ラボのログ・演出制御
