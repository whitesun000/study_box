# direct_info (Project Content Extractor)

プロジェクトのディレクトリ構造とファイルの内容を抽出し、一つのログファイルに統合する開発支援ツールです。GUI版とWebブラウザ版の両方を提供しています。

## 概要
指定したディレクトリ内を再帰的にスキャンし、フォルダ構成のツリー構造とソースコードの内容を一つのテキストファイルとして抽出します。
AI（LLM）へのコンテキスト提供、コードリファクタリング前の現状把握、プロジェクトドキュメントの自動生成に最適です。

## 特徴
- ### マルチインターフェース：
    - **GUI版**：フォルダ選択や保存先の指定をマウス操作で行えます。
    - **Web版**：FastAPIによるブラウザベースの操作
- **共通ロジックの再利用**: 処理の核心（utils）をカプセル化し、UI層のみを切り分けたクリーンな設計。
- **階層構造の可視化**: フォルダ構成をインデント付きのツリー形式で出力します。
- **柔軟な抽出オプション**: ファイルの中身を書き出すかどうかをチェックボックスで切り替え可能です。
- **スマートな除外設定**: `.git`, `__pycache__`, `.venv` などの不要なディレクトリは自動的にスキップします。
- **幅広い対応形式**: `.py`, `.txt`, `.html`, `.css`, `.js`, `.php`, `.md`, `.json` などのテキストファイルを自動抽出します。

## 使い方
1. `main.py` を実行してGUIを起動します。
2. 「スキャン対象フォルダ」を選択します。
3. 「ログ保存先」を確認（または変更）します。
4. ファイルの中身を含める場合はチェックを入れ、「スキャン開始」をクリックします。

## ディレクトリ構成
```text
direct_info/
├── main.py              # GUI版エントリーポイント
├── main_web.py          # Web版エントリーポイント
├── gui/                 # GUI画面構成（Tkinter）
├── web/                 # Web画面構成（FastAPI / HTML / CSS）
│   ├── app.py           # Webアプリロジック
│   ├── static/          # 静的ファイル (CSS)
│   └── templates/       # HTMLテンプレート
└── utils/               # 共通ロジック
    └── file_processor.py # ファイル走査・書き出しの核心部
```
## 使い方
### GUI版
1. ```python main.py``` を実行してGUIを起動します。
2. マウス操作でフォルダと保存先を選択し、「スキャン開始」をクリックします。

### Web版 (Docker推奨)
1. ``` docker compose up -d ```でコンテナを起動します。
2. ブラウザで```http://localhost:8000```にアクセスします。
3. スキャン対象と保存先のパス（コンテナ内パス）を入力し、「スキャン開始」をクリックします。

## 開発環境
- Python 3.12+
- FastAPI/Uvicorn(Web版)
- Jinja2(テンプレートエンジン)
- Tkinter(GUI版)
- Docker/Docker Compose
